基本介绍

在kaggle比赛中经常可以见到GBDT(Gradient Boosting Decision Tree)算法，GBDT属于监督学习方法，广泛应用于分类回归问题。
GBDT有很多马甲，比如 GBRT (Gradient Boosted Regression Tree)，MART1，Boosted Tree等等，见到了要知道他们都是一个人哦。
下面就将GBDT字面拆开，以Boosting => Gradient Boosting => Gradient Boosting Decision Tree的顺序讲解。

Boosting

Boosting算法是一种把几个分类器整合为一个分类器的方法，在其产生前，还出现过两种重要的将多个分类器整合为一个分类器的方法，即boostrapping方法和bagging方法。

Wiki定义：Boosting is a machine learning ensemble meta-algorithm for primarily reducing bias, and also variance in supervised learning, and a family of machine learning algorithms which convert weak learners to strong ones. Boosting is based on the question posed by Kearns and Valiant (1988, 1989):Can a set of weak learners create a single strong learner? A weak learner is defined to be a classifier which is only slightly correlated with the true classification (it can label examples better than random guessing). In contrast, a strong learner is a classifier that is arbitrarily well-correlated with the true classification.

什么是bias，variance？
来自于统计学习里的概念叫做Bias-Variance Tradeoff。Bias可以认为是在有无限多数据时训练模型得到的误差，而Variance则可以认为只有有限数据时，由于随机性所带来的误差。
目标函数 = 误差函数 + 正则项 obj(θ)=L(θ)+Ω(θ)\displaystyle obj(\theta)=L(\theta)+ \Omega(\theta) obj(θ)=L(θ)+Ω(θ) 拿目标函数举例子说明，误差函数让我们的模型尽力去拟合训练数据，以得到尽量小的Bias，而正则项让我们使用更加简单的模型，即简单模型在有限数据拟合后，不容易出现由随机性导致的过拟合，泛化能力更强。
简单来说Boosting算法就是是将弱分类器组合成为强分类器。算法实现中最有名的是Adaboost，核心是在串行训练中迭代更新权重，每次更新提高分错项的权重。
举个机器学习技法课程中的例子帮助理解：学生回答问题，老师负责纠正错误部分，在不断纠正后，达到高的水平。

Gradient Boosting模型

Wiki定义：Gradient boosting is a machine learning technique for regression and classification problems, which produces a prediction model in the form of an ensemble of weak prediction models, typically decision trees. It builds the model in a stage-wise fashion like other boosting methods do, and it generalizes them by allowing optimization of an arbitrary differentiable loss function.

Gradient Boosting的目的就是让损失函数持续下降，每轮迭代先求得关于累加函数FFF的负梯度(- gradient)，然后通过弱学习器fff去拟合该负梯度，再将弱学习器fff累加到累加函数FFF得到新的FFF。
为了便于理解，可将累加函数FFF类比于之前对参数θ\thetaθ做gradient descent，我们如何迭代求解函数f(θ)f(\theta)f(θ)的呢？ 对θ\thetaθ的目标函数求负梯度，然后再和原来的θ\thetaθ累加更新。
Gradient Boosting是在函数空间求函数的负梯度，而之前的Gradient descent是在常数空间求参数的负梯度。

GBDT/GBRT

Gradient Boosting Decision Tree为啥和Gradient Boosting Regression Tree是一个东西呢？因为他俩的区分只是在于Gradient Boosting的目标函数中损失函数不同而已， 他们的基本组成部分叫做回归树(regression tree)，也叫做CART，一个CART往往过于简单无法有效地预测，因此一个更加强力的模型叫做tree ensemble。
以下是loss function及其一阶导数的集合图 常见的随机森林、GBDT和tree ensemble有什么关系呢？
RF和GBDT的模型都是tree ensemble，只是构造模型参数的方法不同，RF是bagging出大量样本、特征，对每份样本特征构造树，GBDT采用additive training的方式，每一次保留原来的模型不变，通过串行迭代加入一个新的函数fff到我们的模型中。每一轮加入什么fff呢？答案很简单，选取的那个fff使得目标函数尽量小，即函数负梯度方向。

优缺点

优：精度高，能做非线性数据。
缺：并行难（串行逻辑，前后依赖关系），多类别分类算法时空复杂度大。

参数介绍

主要参数：迭代次数（树个数），学习速率（正则参数），树深度（叶子节点数）
其他：训练数据采样比例，训练特征采样比例

总结

Gradient Boosting Decision Tree就是：

  1. Gradient–在函数的世界中做Gradient(functional gradient)
  2. Boosting–串行叠加(additive training)
  3. D-Tree–树做弱分类器(tree ensemble)

虽GBDT有众多优点，但因其为串行逻辑，有前后依赖关系，故其算法速度慢，在比赛中数据量达到一定量级后，训练时间难以忍受。这时，XGBoost的出现点亮了世界，详见下回分解。
